# Google Play – Data Safety Form Answers

## Does your app collect or share any of the required user data types?
**Yes** — location data (optional, user-initiated only)

---

## Data types collected

### Location
- **Approximate location:** No
- **Precise location:** Yes
- **Collection:** Optional, only when user taps "Add location" on an entry
- **Purpose:** App functionality (displaying place name in entry)
- **Processed ephemerally:** No — stored in the entry on-device
- **Required or optional:** Optional (user-initiated)
- **Shared with third parties:** No

### No other data types are collected.
The following are NOT collected:
- Personal info (name, email, address, etc.)
- Financial info
- Health and fitness (mood/health ratings are stored only locally, not shared)
- Messages
- Photos or videos
- Audio files
- Files and docs
- Calendar events
- Contacts
- App activity beyond what's stored locally
- Web browsing
- App info and performance
- Device or other IDs

---

## Data sharing
**No user data is shared with third parties.**

The optional Nextcloud sync sends data to a server chosen and operated by the user themselves. This is not considered "sharing with third parties" by Google's definition.

---

## Security practices
- [ ] Data is encrypted in transit — **Yes** (Nextcloud sync uses HTTPS; optional AES encryption of backup file)
- [ ] You provide a way for users to request that their data is deleted — **Yes** (uninstalling the app deletes all local data; users manage their own Nextcloud backup)
